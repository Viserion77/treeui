<Title>Contribution</Title>

Contributions should preserve framework-agnostic naming, strong defaults, and simple public APIs.
When a public behavior changes, update tests and documentation together.

Use Changesets for package releases and follow the Docker-first workflow documented in the repository root.
When preparing a publish-ready branch, add a changeset, run `pnpm version-packages`, and let the release workflow on `main` handle publishing.
